# 🔐 Авторизация API (Authentication System)

## 🎯 Мақсаты:
Бұл жобаның мақсаты — қауіпсіздік негіздерін түсіну және пайдаланушыны тіркеу мен жүйеге кіру логикасын жүзеге асыру.

---

## 📌 Жобаның сипаттамасы:
Бағдарлама пайдаланушыларды тіркеуге (register) және жүйеге кіруге (login) мүмкіндік береді.  
Парольдер қауіпсіздік үшін хэштеу арқылы сақталады.

---

## ❓ Сұрақтар және жауаптар:

### 1. Хэштеу деген не?
Хэштеу — бұл мәліметті (мысалы, парольді) арнайы алгоритм арқылы өзгертіп, оны бастапқы күйіне қайта келтіру мүмкін емес форматқа айналдыру процесі.

---

### 2. Неге парольді ашық сақтауға болмайды?
Егер пароль ашық түрде сақталса, деректер базасы бұзылған жағдайда барлық пайдаланушылардың құпия мәліметтері қолжетімді болады. Хэштеу бұл қауіптің алдын алады.

---

### 3. Авторизация қалай жұмыс істейді?
Авторизация келесі қадамдардан тұрады:
- Пайдаланушы тіркеледі (логин + пароль)  
- Пароль хэш түрінде сақталады  
- Кіру кезінде енгізілген пароль қайта хэштенеді  
- Егер хэштер сәйкес келсе — жүйеге кіруге рұқсат беріледі  

---

## 📊 Қорытынды:
Бұл жоба қауіпсіздік негіздерін түсінуге көмектеседі. Парольдерді хэштеу арқылы деректерді қорғауға болады. Авторизация жүйесі кез келген қосымшада маңызды рөл атқарады.
